# 3D_Project
3D 포트폴리오 프로젝트

###아이템 파라미터 서순
(불러올 스크립트 네임)_(스킬 프리펩 인덱스)


# 1일차

    - 플레이어 이동 (w, a, s, d)구현
        * 카메라 좌표계로 이동하도록 입력을 컨버팅해줌

    - 플레이어 캐릭터가 마우스를 바라보도록 제작

    - 이에 따라 입력과 더불어 마우스 위치랑 바라보는 방향을 통해 애니메이션 제작
        * 컨버팅 해준 입력을 다시 로컬의 입력으로 변환해 블랜드 트리의 입력으로 사용

# 2일차

    - 컴벳 시스템으로 플레이어가 공격할 때 데미지가 들어가게 만듬
        * 공격시에는 이동과 회전이 불가
        * 공격시에만 무기 콜라이더 On

    - 웨폰 시스템을 통해 무기 스탯을 적용
        * 공격 속도는 애니메이션 속도를 조절하는 형태로 구현

    - 플레이어 스탯 구현
        * 아직 구현한 스탯 체력 밖에 없음. 패시브 트리 만들면 그 때 추가 할 예정

    - 몬스터의 인식 범위 내에 플레이어가 존재하면 플레이어를 쫓아오도록 구현
        * 네비게이션 매쉬 사용

    - 몬스터의 정지, 이동, 공격, 사망 애니메이션 구현 및 적용

# 3일차

    - 스킬 시스템 구조 이해
    - 이해한 구조를 통해 스킬 프로토타입 제작(아직 미완성이라 UI 완성 후 추후 완성)

# 4일차

    - 인벤토리 구조의 이해

    - 이해한 구조를 통해 인벤토리와 스킬젬탭 제작

    - 아이템 구조화
        * 아이템 테이블.csv를 만들고 외부에서 데이터를 관리할 수 있도록 함
        * 내부에서 테이블을 조회하고 아이템을 서치할 수 있게 함
        * 스크립터블 오브젝트를 활용해 아이템 추가를 쉽게 함

# 5일차

    - 젬 시스템 구조화 완성

    - UI에 젬을 뺏다 꼈다 하면 기능이 동작되게 함 (젬 시스템과 UI 시스템의 연동)

    - 이를 통해 스킬 시스템, 젬 시스템, UI 시스템을 모두 연동

    - FireBall 젬과 스킬(데미지와 컴벳 시스템과의 연동) 구현

    - 스킬 애니메이션 구현 : 시전 모션과 시전 위치 등 구현

# 6일차

    - 플레이어 UI 제작 및 플레이어 스탯과 연동

    - 전투 시스템 업데이트
        * 플레이어나 몬스터가 맞으면 피가 튀도록 만듬
        * 몬스터를 죽이면 경험치를 얻도록 구현

    - 스킬 시스템 업데이트
        * 스킬 쓰면 마나가 닳도록 함(근데 구조 수정은 해야 할 듯)

    - 마우스 커서를 몬스터에게 가져가면 체력바가 표시되게 함

    - 인벤토리에 장비 슬롯 만들고 드래그 슬롯 사이즈가 장비에 따라 바뀌도록 만듬

# 7일차

    - 장비 시스템 구조화 및 갑옷 하나 만듬
